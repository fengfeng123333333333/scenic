# 自然生态类景区小程序设计规范

## 一、总体设计原则（Design Principles）

- **核心理念**：以"自然、清新、有氧、回归自然"为核心，传达亲近自然、舒缓与可信任的视觉感受。
- **简洁优先**：保留信息层级，避免视觉噪音，使用柔和的自然色调而非鲜艳人工色。
- **层次清晰**：用色块、阴影、间距与字重区分信息层级，使用户快速找到关键操作（购票/查询）。
- **触感优先**：控件与交互元素需满足移动端最小触控面积并提供反馈（缩放/颜色变化）。
- **统一与可扩展**：设计 token（颜色/间距/圆角/阴影）应可复用于后续页面和组件库。
- **适配机型**：样式严格遵循设计规范中的配色、字体、间距、圆角、阴影体系，但必须适配各种手机屏幕宽度（以 390px 设计稿为基准，使用相对单位进行适配）。

## 二、整体色彩系统（Color System）

**目标**：营造自然生态氛围的同时保证信息对比与可访问性。

### 主色系（Primary）

- 主色（Leaf Green）`#3AA76D` — 用途：主 CTA、激活态、重要图标、选中态。
- 主色深 `#2E8B57`（用于边界或深色场景）

### 辅色（Secondary / Accent）

- 湿地蓝 `#4EA6C8` — 用途：次要强调、辅助按钮、标签、图形装饰。
- 暖土色 `#D8B48A`（可选）— 用于点缀（不常用），例如活动角标。

### 中性色（Neutral）

- 页面背景 `#F6FBF7`（非常浅的绿白）
- 卡片背景 / 面板 `#FFFFFF`
- 主文本 `#2F3A35`（深自然灰）
- 次文本 `#6B766F`（中灰）
- 轻分割 / 边线 `#E6EFE8`
- 失效 / 禁用 `#C9D9D0`

### 强调色（Warning / Success / Info）

- 成功绿 `#38A169`（或使用主色深）
- 警示橙 `#F6A623`（慎用）
- 信息蓝 `#3B82F6`（仅限通用信息，不常用）

### 可访问性提示

- 主文本与背景对比度需 >= 4.5:1（关键文案）
- 交互颜色（主色）在白底应显著且可被色盲用户区分（可通过图标或形状辅助）

## 三、字体系统（Typography）

**目标**：移动端可读、层次明确、视觉温和。

### 字体族（Fallback 列表）

首选系统无衬线：`PingFang SC / Roboto / system-ui, -apple-system, "Helvetica Neue", Arial`

### 字号与权重（移动端推荐）

| 层级                       | 字号     | 字重           | 行高  |
| -------------------------- | -------- | -------------- | ----- |
| Title / H1（页面主要标题） | 16–18 px | 600 (Semibold) | 22 px |
| H2（模块标题）             | 15 px    | 600            | 20 px |
| Body（正文）               | 14 px    | 400            | 20 px |
| Caption（说明/次级）       | 12–13 px | 400            | 18 px |
| Button 文本                | 14–16 px | 600            | —     |
| 小字（非常次要）           | 11–12 px | —              | —     |

- 标题与 CTA 使用 Semibold（600）以强化视觉焦点
- 正文使用 Regular（400）保证可读性
- 单段宽度控制在 40–70 字符，段间距建议为 8–12 px

## 四、间距与栅格（Spacing & Grid）

**目标**：构建一致的垂直节奏与水平容器限制。

### 基础尺度

- 基础单位：4 px（scale: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64）
- 页面左右内边距（gutter）：16 px（移动端标准）
- 模块间距（常规）：24 px（`--space-6`）
- 模块内边距：12–16 px（卡片内边距推荐 12–16 px）
- 元素最小触控区：48 x 48 px（优先）

### 栅格

- 宽度基准：设计稿基于 390 px 宽
- 列布局：多数模块使用全宽流式布局；图标网格使用 4 列，商品卡使用 2 列网格（每列留间距 gap 12–16 px）
- 栅格示例：`grid-template-columns: repeat(2, 1fr); gap: 12–16 px`

## 五、圆角 / 边框 / 阴影（Radius / Borders / Shadows）

**目标**：温和的圆角与浅色阴影增强浮层与层级感（自然、轻盈）。

### 圆角

- 全局卡片 radius: 12–16 px（推荐 14 px）
- 按钮 radius: 10–14 px（推荐 12 px）
- 图标容器 radius: 50%（圆形）或 28 px（直径 56 px 圆）
- 小控件 radius: 8–10 px（如 badge、标签）

### 边框

- 尽量减少 1px 深边框，优先用背景差与阴影分层
- 若需边框：使用非常浅的绿色系 `#E6EFE8` 或 `1px rgba(0,0,0,0.04)`

### 阴影

- 统一轻柔绿色调阴影，避免浓重黑阴影
- 卡片、浮层示例：`box-shadow: 0 6px 16px rgba(46,139,87,0.06)`
- 交互阴影（按压/悬浮）：按压时阴影减弱或去除（移动端用按压缩放替代 hover）

## 六、按钮（Button）

### 6.1 类型与语义

| 类型      | 用途                           | 视觉特征                                                                               |
| --------- | ------------------------------ | -------------------------------------------------------------------------------------- |
| primary   | 主要 CTA（立即支付、确认购买） | 实心填充 `var(--color-primary)`，文字 `var(--color-text-on-primary)`                   |
| secondary | 次要操作（查看详情、取消）     | 透明背景 + 1px 边框 `var(--color-primary-border-light)`，文字色 `var(--color-primary)` |
| ghost     | 非破坏性辅助（查看更多）       | 无边框/背景，文字色 `var(--color-primary)`                                             |
| danger    | 警示/删除操作                  | 边框或背景使用 `--color-danger`，配合删除图标                                          |

### 6.2 尺寸

- **Large**：高度 `var(--btn-height-lg)`，字号 `var(--btn-font-lg)`，内边距 `var(--btn-padding-lg)`
- **Medium**（默认）：高度 `var(--btn-height-md)`，字号 `var(--btn-font-md)`，内边距 `var(--btn-padding-md)`
- **Small**：高度 `var(--btn-height-sm)`，字号 `var(--btn-font-sm)`，内边距 `var(--btn-padding-sm)`

### 6.3 图标按钮

- **入口型**（首页功能入口）：直径 `var(--btn-icon-entry)`，圆形，内部图标尺寸 `var(--btn-icon-size-lg)`，线宽 2px
- **工具栏型**（导航栏/列表操作）：直径 `var(--btn-icon-toolbar)`，内部图标尺寸 `var(--btn-icon-size-md)`

### 6.4 状态（完整交互闭环）

所有按钮必须覆盖以下状态：

| 状态            | 表现                                                                                        | 实现要点                                          |
| --------------- | ------------------------------------------------------------------------------------------- | ------------------------------------------------- |
| Normal          | 对应类型的视觉                                                                              | —                                                 |
| Hover（桌面端） | Primary 背景变 `--color-primary-hover`；Secondary/Ghost 背景变为 `--color-primary-bg-light` | 移动端可省略，但组件应预留                        |
| Focus-visible   | `box-shadow: 0 0 0 4px var(--color-primary-border-light)`；`outline: none`                  | 键盘导航时可见                                    |
| Active / Press  | `transform: scale(0.96)`，阴影减弱或消失                                                    | 120ms 过渡                                        |
| Loading         | 按钮内显示 16px spinner，禁用点击，透明度 0.9                                               | 保留文字或仅显示 spinner；设置 `aria-busy="true"` |
| Disabled        | `opacity: 0.45`，`cursor: not-allowed`，不响应事件                                          | 禁用所有交互                                      |

### 6.6 组件 API

```ts
type ButtonType = "primary" | "secondary" | "ghost" | "danger";
type ButtonSize = "small" | "medium" | "large";

interface ButtonProps {
  type?: ButtonType;
  size?: ButtonSize;
  loading?: boolean;
  disabled?: boolean;
  icon?: string | ReactNode; // 图标名称或组件
  children?: ReactNode;
  onClick?: () => void;
  ariaLabel?: string; // 纯图标按钮必须提供
}
```

## 七、加载（Loading）

### 7.1 场景分类

| 场景                 | 使用方式                                                                          | 尺寸                |
| -------------------- | --------------------------------------------------------------------------------- | ------------------- |
| 页面级（全屏遮罩）   | 居中 spinner + 可选文字，半透明遮罩 `rgba(0,0,0,0.04)` 或 `rgba(255,255,255,0.8)` | `var(--spinner-lg)` |
| 区块级（局部内容区） | 优先使用骨架屏；其次在区域中心放置 spinner                                        | `var(--spinner-md)` |
| 控件级（按钮内）     | 与文字并列或替换文字，按钮禁用                                                    | `var(--spinner-sm)` |

### 7.2 Spinner 动画

```css
.spinner {
  border-radius: 50%;
  animation: spin var(--spinner-duration) linear infinite;
}
@keyframes spin {
  to {
    transform: rotate(360deg);
  }
}

/* 颜色变体：深色背景用白色 */
.spinner-sm-white {
  width: var(--spinner-sm);
  height: var(--spinner-sm);
  border: 2px solid rgba(255, 255, 255, 0.25);
  border-top-color: #fff;
}
/* 浅色背景用主色 */
.spinner-sm-primary {
  width: var(--spinner-sm);
  height: var(--spinner-sm);
  border: 2px solid var(--color-primary-border-light);
  border-top-color: var(--color-primary);
}
/* 区块级 */
.spinner-md {
  width: var(--spinner-md);
  height: var(--spinner-md);
  border: 3px solid var(--color-primary-border-light);
  border-top-color: var(--color-primary);
}
/* 页面级 */
.spinner-lg {
  width: var(--spinner-lg);
  height: var(--spinner-lg);
  border: 4px solid var(--color-primary-border-light);
  border-top-color: var(--color-primary);
}
```

### 7.3 延迟展示策略（防闪烁）

- 请求发起后 **300ms 内**不展示 loading。
- 超过 **300ms** 时显示 spinner / skeleton。
- 加载状态最少保持 **400～600ms**，即使提前完成也保留动画过渡。
- 超过 **5 秒**提示超时，并显示重试按钮。

实现参考：

```js
const [showLoader, setShowLoader] = useState(false);
useEffect(() => {
  const timer = setTimeout(() => setShowLoader(true), 300);
  fetchData().finally(() => {
    clearTimeout(timer);
    setShowLoader(false);
  });
}, []);
```

### 7.4 可访问性

- 页面级 spinner 容器设置 `aria-live="polite"`，并包含 visually-hidden 文字"正在加载…"
- 按钮加载时添加 `aria-label="加载中"` 或 `aria-busy="true"`
- 为加载中的元素禁用焦点和交互

## 八、骨架屏（Skeleton）

### 8.1 使用原则

- 每个存在异步数据加载的页面必须配套骨架屏组件，存放于 `components/skeleton/` 目录。
- 命名规则：`<页面名>_skeleton.vue`（例如首页 `index.vue` 对应 `index_skeleton.vue`）。
- 页面进入时立即展示骨架屏，所有接口请求完成后（成功或失败）切换为真实内容。
- 若无接口请求，展示 **600～800ms** 后淡出消失。
- 骨架消失时添加 **240ms** `opacity` 淡出过渡，防止布局跳动。

### 8.2 视觉规范

- **底色**：`var(--color-skeleton-base)`（`#E6EFE8`）
- **Shimmer 动画**：

```css
.skeleton {
  background: linear-gradient(
    90deg,
    var(--color-skeleton-base) 0%,
    var(--color-skeleton-highlight) 50%,
    var(--color-skeleton-base) 100%
  );
  background-size: 200% 100%;
  animation: shimmer var(--skeleton-duration) ease-in-out infinite;
  border-radius: var(--skeleton-radius);
}
@keyframes shimmer {
  0% {
    background-position: -200% 0;
  }
  100% {
    background-position: 200% 0;
  }
}
```

- **圆角**：文本条用 8px，圆形元素 50%，卡片区域 14px（或 `var(--radius-card)`）

### 8.3 常见组合模板

**商品卡片骨架：**

```html
<div class="product-skeleton">
  <div
    class="skeleton thumb"
    style="aspect-ratio:4/3; border-radius:14px;"
  ></div>
  <div
    class="skeleton title"
    style="height:16px; width:70%; margin-top:12px; border-radius:8px;"
  ></div>
  <div
    class="skeleton meta"
    style="height:12px; width:45%; margin-top:8px; border-radius:8px;"
  ></div>
  <div
    class="skeleton price"
    style="height:16px; width:30%; margin-top:8px; border-radius:8px;"
  ></div>
</div>
```

**图标网格骨架：**

- 圆形骨架 `width:56px; height:56px; border-radius:50%;`
- 下方文字条 `width:60%; height:12px; margin-top:8px;`

### 8.4 组件化

提供 BaseSkeleton 组件，支持属性：

```ts
interface SkeletonProps {
  width?: string | number;
  height?: string | number;
  borderRadius?: string;
  variant?: "text" | "circle" | "rect";
}
```

页面级骨架屏组件应继承该组件，组装成与真实页面一致的布局。

### 8.5 注意事项

- 骨架屏仅占位，不可交互，添加 `aria-hidden="true"` 避免屏幕阅读器干扰。
- 真实内容加载完成后立即替换，可配合淡入动画（opacity 0→1，200ms）。
- 骨架元素尺寸必须与最终内容完全一致，避免布局跳动。

## 九、图标与插画规范（Icon & Illustration）

**目标**：使用统一、可识别且与自然主题相契合的图形语言。

### 图标（Icon）

- 风格：圆润线性 + 填充混合（线条圆端 / 圆头），线宽约 2 px
- 颜色：常态主色或深灰；背景容器浅主色透明（例如 `rgba(58,167,109,0.06)`）
- 容器：推荐圆形背景，直径 56 px（首页）或 40 px（工具栏）
- 语义：保持图标直观（门票、地图、购物车、会员、路线、停车等）
- 文件格式：SVG（便于可伸缩与颜色替换）
- 输出：提供 1x / 2x / 3x 的导出或直接提供 SVG

### 插画（Illustration）

- 风格建议：简洁平面或半扁平插画，使用主/辅色与少量暖土色点缀
- 场景元素：山、树、湿地波纹、叶脉、日出/晚霞的柔和色块
- 用法：用于活动横幅、空状态页、指引页等，不用于信息密集区
- 透明度与纹理：可在背景使用非常低透明度（5–8%）的自然纹理（叶脉、水波）作为点缀

## 十、文案风格与排版预期（Copy Tone & Expectation）

- **总体语气**：亲切、自然、简洁、实用。避免过度商业化与生硬表达。
- **优先使用行动导向的短语**（例如："立即购票"、"查看路线"、"预约参观"）
- **标题类**（CTA）：动词优先，短而有力，例如 "购票"、"查看演艺"
- **信息类**：精确，格式统一，例如 "开放时间：09:00–19:00（周一闭馆）"
- **价格/促销**：显著显示现价，原价以较小字号或灰色并带删除线展示；明确标注有效期/限制
- **空状态/错误信息**：友好且提供下一步，例如 "当前暂无可售票项。你可以返回首页或联系客服。"
- **行文长度**：标题 ≤ 12 字（中文），副标题 ≤ 18 字，说明文 ≤ 40–80 字（视模块而定）

## 十一、交互与动效原则（Interaction & Motion）

- 动效短且自然：200–400 ms，缓入缓出（ease-in-out）
- 优先使用微交互（按压反馈、颜色过渡、位移 2–4 px）提升触感
- 禁用态与加载态明确（骨架屏或 spinner）

## 十二、主题切换与变量注入

所有颜色和尺寸均通过 CSS 变量定义，组件内部使用 `var(--xxx)` 引用。

主题配置从接口获取（如 `GET /api/theme/nature`），返回 tokens JSON 对象。

前端注入函数：

```javascript
async function applyTheme(name) {
  const { tokens } = await fetch(`/api/theme/${name}`).then((r) => r.json());
  const root = document.documentElement;
  Object.entries(tokens).forEach(([k, v]) => root.style.setProperty(k, v));
}
```

更换主题时仅修改变量值，无需修改组件样式。

## 十三、Token 与变量建议（Design Tokens）

建议输出为供开发使用的 token（JSON 或 CSS 变量）：

- 颜色：`--color-primary`, `--color-secondary`, `--color-bg`, `--color-card`, `--color-text`, `--color-muted`

## 十四、可访问性检查清单（Accessibility）

- [ ] 所有按钮最小触控区域 ≥ 44×44 px（建议 48×48）
- [ ] 主按钮文字与背景对比度 ≥ 4.5:1
- [ ] 纯图标按钮提供 `aria-label` 属性
- [ ] 加载状态延迟 300ms 展示，避免闪烁
- [ ] 按钮 loading 状态设置 `aria-busy="true"` 并禁用点击
- [ ] 骨架屏设置 `aria-hidden="true"`，页面级 spinner 提供 `aria-live="polite"` 区域
- [ ] 键盘导航时所有按钮具备聚焦环（`:focus-visible`）
- [ ] 文字对比度：主要文本与背景对比度 >= 4.5:1
- [ ] 支持高对比模式与放大（通过 rem 或可缩放单位）
- [ ] 图像提供 alt 文本；按钮/链接提供 aria-label（Web 或小程序无障碍同理）
