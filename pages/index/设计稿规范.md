# 自然生态类景区小程序设计规范

## 一、总体设计原则（Design Principles）

- **核心理念**：以“自然、清新、有氧、回归自然”为核心，传达亲近自然、舒缓与可信任的视觉感受。
- **简洁优先**：保留信息层级，避免视觉噪音，使用柔和的自然色调而非鲜艳人工色。
- **层次清晰**：用色块、阴影、间距与字重区分信息层级，使用户快速找到关键操作（购票/查询）。
- **触感优先**：控件与交互元素需满足移动端最小触控面积并提供反馈（缩放/颜色变化）。
- **统一与可扩展**：设计 token（颜色/间距/圆角/阴影）应可复用于后续页面和组件库。
- **适配机型**：样式严格遵循设计规范中的配色、字体、间距、圆角、阴影体系，但必须适配各种手机屏幕宽度（以 390px 设计稿为基准，使用相对单位进行适配）。

## 二、整体色彩系统（Color System）

**目标**：营造自然生态氛围的同时保证信息对比与可访问性。

### 主色系（Primary）

- 主色（Leaf Green）`#3AA76D` — 用途：主 CTA、激活态、重要图标、选中态。
- 主色深 `#2E8B57`（用于边界或深色场景）

### 辅色（Secondary / Accent）

- 湿地蓝 `#4EA6C8` — 用途：次要强调、辅助按钮、标签、图形装饰。
- 暖土色 `#D8B48A`（可选）— 用于点缀（不常用），例如活动角标。

### 中性色（Neutral）

- 页面背景 `#F6FBF7`（非常浅的绿白）
- 卡片背景 / 面板 `#FFFFFF`
- 主文本 `#2F3A35`（深自然灰）
- 次文本 `#6B766F`（中灰）
- 轻分割 / 边线 `#E6EFE8`
- 失效 / 禁用 `#C9D9D0`

### 强调色（Warning / Success / Info）

- 成功绿 `#38A169`（或使用主色深）
- 警示橙 `#F6A623`（慎用）
- 信息蓝 `#3B82F6`（仅限通用信息，不常用）

### 可访问性提示

- 主文本与背景对比度需 >= 4.5:1（关键文案）
- 交互颜色（主色）在白底应显著且可被色盲用户区分（可通过图标或形状辅助）

## 三、字体系统（Typography）

**目标**：移动端可读、层次明确、视觉温和。

### 字体族（Fallback 列表）

首选系统无衬线：`PingFang SC / Roboto / system-ui, -apple-system, "Helvetica Neue", Arial`

### 字号与权重（移动端推荐）

| 层级                       | 字号     | 字重           | 行高  |
| -------------------------- | -------- | -------------- | ----- |
| Title / H1（页面主要标题） | 16–18 px | 600 (Semibold) | 22 px |
| H2（模块标题）             | 15 px    | 600            | 20 px |
| Body（正文）               | 14 px    | 400            | 20 px |
| Caption（说明/次级）       | 12–13 px | 400            | 18 px |
| Button 文本                | 14–16 px | 600            | —     |
| 小字（非常次要）           | 11–12 px | —              | —     |

- 标题与 CTA 使用 Semibold（600）以强化视觉焦点
- 正文使用 Regular（400）保证可读性
- 单段宽度控制在 40–70 字符，段间距建议为 8–12 px

## 四、间距与栅格（Spacing & Grid）

**目标**：构建一致的垂直节奏与水平容器限制。

### 基础尺度

- 基础单位：4 px（scale: 4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64）
- 页面左右内边距（gutter）：16 px（移动端标准）
- 模块间距（常规）：24 px（`--space-6`）
- 模块内边距：12–16 px（卡片内边距推荐 12–16 px）
- 元素最小触控区：48 x 48 px（优先）

### 栅格

- 宽度基准：设计稿基于 390 px 宽
- 列布局：多数模块使用全宽流式布局；图标网格使用 4 列，商品卡使用 2 列网格（每列留间距 gap 12–16 px）
- 栅格示例：`grid-template-columns: repeat(2, 1fr); gap: 12–16 px`

## 五、圆角 / 边框 / 阴影（Radius / Borders / Shadows）

**目标**：温和的圆角与浅色阴影增强浮层与层级感（自然、轻盈）。

### 圆角

- 全局卡片 radius: 12–16 px（推荐 14 px）
- 按钮 radius: 10–14 px（推荐 12 px）
- 图标容器 radius: 50%（圆形）或 28 px（直径 56 px 圆）
- 小控件 radius: 8–10 px（如 badge、标签）

### 边框

- 尽量减少 1px 深边框，优先用背景差与阴影分层
- 若需边框：使用非常浅的绿色系 `#E6EFE8` 或 `1px rgba(0,0,0,0.04)`

### 阴影

- 统一轻柔绿色调阴影，避免浓重黑阴影
- 卡片、浮层示例：`box-shadow: 0 6px 16px rgba(46,139,87,0.06)`
- 交互阴影（按压/悬浮）：按压时阴影减弱或去除（移动端用按压缩放替代 hover）

## 六、核心页面模块设计规范（逐模块详述）

### 模块 1 — 顶部安全区（Safe Area）

- **目的**：与系统状态栏衔接，避免内容被刘海或状态栏覆盖
- **尺寸**：高度依设备而定，建议保留 12–16 px 视觉留白
- **样式**：透明或与背景一致，不含复杂视觉元素
- **可复用性**：所有页面通用，使用 `env(safe-area-inset-top)`

### 模块 2 — 头部（Header，当前被移除或极简）

- **目的**：页面标识与全局操作（返回、搜索、更多）
- **尺寸**：高度 48–56 px，左右 padding 16 px
- **元素**：左图标（返回/主页）、中标题（15–16 px semibold）、右图标（搜索/更多）
- **样式**：背景可透明或卡片白，图标使用主色或深灰
- **交互**：图标点击提供 120ms 缩放反馈（scale 0.96）

### 模块 3 — 轮播图（Hero / Carousel）

- **目的**：视觉焦点、推广活动、重要信息的承载
- **尺寸**：高度约为屏宽 \* 0.38–0.44（在 390 宽下约 150–170 px）
- **图片处理**：保留原图内容，不替换；对全图做亮度与饱和度微调（亮度 +5%，饱和度 -8% 建议值）
- **遮罩**：使用柔和渐变覆盖（顶部透明到底部浅绿色叠加），确保文字可读  
  示例：`linear-gradient(180deg, rgba(255,255,255,0) 40%, rgba(232,246,236,0.6) 100%)`
- **文本叠加**：置于遮罩高对比部位（底部/左下），字体 15px semibold，文字阴影 `0 4px 10px rgba(0,0,0,0.25)`
- **指示器**：居中底部的小圆点（6–8 px），active 用主色 `#3AA76D`，inactive 半透明白或浅绿描边
- **交互**：自动轮播、手势滑动、分页点点击，切换动画 300–400 ms ease-in-out

### 模块 4 — 信息卡（Open Info）

- **目的**：展示开放时间 / 地址 / 联系电话等要点信息
- **布局**：单列卡片，内边距 12–16 px，卡片圆角 14 px，box-shadow 轻柔
- **文案**：label（13px 次色）+ value（14 px 主色），label 固定宽度如 72 px 以对齐视觉
- **颜色**：卡片白背景 `#FFFFFF`，label 使用 `#6B766F`，value 使用 `#2F3A35`

### 模块 5 — 功能图标网格（Icon Grid）

- **目的**：提供高频入口（购票/地图/导游/会员）
- **布局**：4 列网格（`grid-template-columns: repeat(4, 1fr)`），gap 12–16 px
- **图标容器**：圆形背景，直径 56 px，背景色为主色低透明（如 `rgba(58,167,109,0.06)`）或白色
- **图标样式**：线性+填充，线宽 ~2 px，颜色主色或深灰
- **标签**：12 px 次文本，居中
- **交互**：点击反馈 scale 0.96，active 可切换至主色背景

### 模块 6 — 模块切换 / Tab 导航

- **目的**：切换商品类型（景区门票 / 商品套餐 / 会员充值）
- **布局**：水平分布，居中，容器高度 48 px，卡片背景或白底
- **选中样式**：文字主色 `#3AA76D`，下方指示条 2–4 px 或下划线效果
- **文本**：15 px semibold

### 模块 7 — 商品/票务卡片列表（Product Grid）

- **目的**：展示门票与套餐，支持跳转购买
- **布局**：2 列网格，gap 12–16 px；卡片圆角 14 px，box-shadow 轻柔
- **图片**：比例 4:3（列表），object-fit: cover，顶部或整张图做圆角裁剪
- **文字**：标题 14 px semibold；价格 16 px bold，主色 `#3AA76D`
- **价格展示**：当前价格高亮，原价使用灰色并带删除线或降低透明度
- **交互**：卡片点击进入详情，点击动画 160–240 ms（轻微缩放 + 阴影变化）

### 模块 8 — 页脚信息（Footer）

- **目的**：显示技术支持、版本号、版权等次要信息
- **布局**：居中，12 px 字体，颜色 `#6B766F`，间距 24–32 px 上方
- **可复用性**：通用页脚组件

### 模块 9 — 底部 Tab 栏（Bottom Navigation）

- **目的**：主导航，四项（首页 / 订单 / 购票 / 我的）
- **布局**：固定底部，height 56–64 px（含安全区），背景白或卡片白，顶部有轻分割或阴影
- **图标**：未选灰色 `#6B766F`，选中主色 `#3AA76D`，文本 12–13 px
- **交互**：点击切换页面并给出 120 ms 缩放或色彩反馈

### 模块 10 — 全局浮层（Modal / Toast）

- **目的**：确认、提示、加载使用
- **样式**：背景白，radius 14 px，box-shadow 深一点（`0 12px 30px rgba(0,0,0,0.12)`），内边距 16 px
- **文案**：标题 15–16 px，正文 14 px，操作按钮在底部水平排列

## 七、核心组件（Component Library）与状态（States）

**目标**：明确定义每个核心组件的视觉状态，便于前端实现。

### 按钮（Button）

- **类型**：Primary（实底主色）、Secondary（边框/透明）、Ghost（透明无边框）
- **尺寸**：Default min-height 44–48 px，padding 10px 16px
- **圆角**：12 px
- **字体**：14–16 px semibold
- **状态**：
  - 常态：Primary 背景 `#3AA76D`，文字白；Secondary 边框 `1px rgba(58,167,109,0.12)`
  - Hover：略亮或阴影加深（移动端通常无 hover）
  - Active / Press：scale 0.96，阴影减弱或无阴影
  - Disabled：background `#C9D9D0`，opacity 0.6，禁止点击

### 卡片（Card）

- 圆角 14 px，box-shadow `var(--shadow-soft)`，背景白
- 状态：normal / pressed（按压缩小 2–4 px 并减弱阴影） / loading（骨架屏）

### 输入框（Input）

- 背景白或非常浅色，圆角 10 px，边框 1 px `#E6EFE8`（无聚焦）
- 聚焦态：边框或下侧线使用主色 `#3AA76D`，box-shadow 轻微内外光晕
- 占位符使用次级文本色 `#6B766F`

### 图标按钮（Icon Button）

- 容器：56 px 圆形，背景浅色，图标 28 px
- 状态：normal / hover（移动端用 active press） / disabled（背景灰）

### 轮播（Carousel）

- 见“模块 3”详述
- 状态：auto-play / paused / swipe / loading

### Tab（局部/全局）

- Tab active: 字体主色 + 底部指示条
- Tab inactive: 次级灰

## 八、图标与插画规范（Icon & Illustration）

**目标**：使用统一、可识别且与自然主题相契合的图形语言。

### 图标（Icon）

- 风格：圆润线性 + 填充混合（线条圆端 / 圆头），线宽约 2 px
- 颜色：常态主色或深灰；背景容器浅主色透明（例如 `rgba(58,167,109,0.06)`）
- 容器：推荐圆形背景，直径 56 px（首页）或 40 px（工具栏）
- 语义：保持图标直观（门票、地图、购物车、会员、路线、停车等）
- 文件格式：SVG（便于可伸缩与颜色替换）
- 输出：提供 1x / 2x / 3x 的导出或直接提供 SVG

### 插画（Illustration）

- 风格建议：简洁平面或半扁平插画，使用主/辅色与少量暖土色点缀
- 场景元素：山、树、湿地波纹、叶脉、日出/晚霞的柔和色块
- 用法：用于活动横幅、空状态页、指引页等，不用于信息密集区
- 透明度与纹理：可在背景使用非常低透明度（5–8%）的自然纹理（叶脉、水波）作为点缀

## 九、文案风格与排版预期（Copy Tone & Expectation）

- **总体语气**：亲切、自然、简洁、实用。避免过度商业化与生硬表达。
- **优先使用行动导向的短语**（例如：“立即购票”、“查看路线”、“预约参观”）
- **标题类**（CTA）：动词优先，短而有力，例如 “购票”、“查看演艺”
- **信息类**：精确，格式统一，例如 “开放时间：09:00–19:00（周一闭馆）”
- **价格/促销**：显著显示现价，原价以较小字号或灰色并带删除线展示；明确标注有效期/限制
- **空状态/错误信息**：友好且提供下一步，例如 “当前暂无可售票项。你可以返回首页或联系客服。”
- **行文长度**：标题 ≤ 12 字（中文），副标题 ≤ 18 字，说明文 ≤ 40–80 字（视模块而定）

## 十、可访问性（Accessibility）

- 文字对比度：主要文本与背景对比度 >= 4.5:1
- 点击区域：最小 44x44 px，建议 48x48 px
- 支持高对比模式与放大（通过 rem 或可缩放单位）
- 图像提供 alt 文本；按钮/链接提供 aria-label（Web 或小程序无障碍同理）

## 十一、交互与动效原则（Interaction & Motion）

- 动效短且自然：200–400 ms，缓入缓出（ease-in-out）
- 优先使用微交互（按压反馈、颜色过渡、位移 2–4 px）提升触感
- 轮播切换：300–400 ms，带淡入淡出或位移缓动
- 禁用态与加载态明确（骨架屏或 spinner）

## 十二、骨架屏规范（Skeleton Screen）

**目标**：在页面 API 请求期间提供即时视觉反馈，降低用户感知等待时间，避免白屏或空白区域。

### 适用范围

- 每个页面必须配套一个独立的骨架屏组件，统一放置于 `components/skeleton/` 目录下
- 命名规则：`<页面名>_skeleton.vue`（例如首页 `index.vue` 对应 `index_skeleton.vue`）
- 页面中按相对路径引入：`import IndexSkeleton from "../../components/skeleton/index_skeleton.vue"`

### 行为规则

1. **有 API 请求时**：页面进入时立即展示骨架屏，所有 API 请求完成后（不论成功或失败）切换为真实内容
2. **无 API 请求时**：骨架屏展示后延时 600–800 ms 自动消失，确保用户感知到加载反馈

### 视觉规范

- **底色**：使用 `--color-border-light`（`#E6EFE8`）作为骨架占位色
- **微光扫过动画**：`linear-gradient(90deg, transparent → rgba(255,255,255,0.5) → rgba(255,255,255,0.55) → rgba(255,255,255,0.5) → transparent)`
- **动画时长**：1.5 s，`ease-in-out`，`infinite`
- **布局匹配**：骨架屏中所有占位块应对齐页面真实模块的位置和尺寸，做到无跳变切换
- **圆角**：占位条使用 `--radius-sm`（8 px → 16 rpx），图标占位使用圆形，卡片使用 `--radius-card`

### 组件接口

```ts
// 骨架屏组件为纯展示型，无 props / emits
// 由父页面通过 v-if 控制显隐
<IndexSkeleton v-if="!pageReady" />
```

### 切换过渡（推荐）

骨架屏消失时可配合 200–300 ms 的 `opacity` 淡出过渡，使切换更平滑：

```css
.skeleton-page {
  transition: opacity 240ms ease-in-out;
}
.skeleton-page.is-hidden {
  opacity: 0;
  pointer-events: none;
}
```

## 十三、Token 与变量建议（Design Tokens）

建议输出为供开发使用的 token（JSON 或 CSS 变量）：

- 颜色：`--color-primary`, `--color-secondary`, `--color-bg`, `--color-card`, `--color-text`, `--color-muted`
- 圆角：`--radius-card` (14px), `--radius-button` (12px)
- 间距：`--space-1`..`--space-8`（4px..32px）
- 阴影：`--shadow-soft`
- 字体：`--font-size-h1`, `--font-size-body`, `--font-size-caption`

### 示例 CSS 变量

```css
:root {
  --color-primary: #3aa76d;
  --color-secondary: #4ea6c8;
  --color-bg: #f6fbf7;
  --color-card: #ffffff;
  --color-text: #2f3a35;
  --radius-card: 14px;
  --space-4: 16px;
  --shadow-soft: 0 6px 16px rgba(46, 139, 87, 0.06);
  /* 更多变量按需定义 */
}
```
